사용자는 PM에게 작업 위임 후 "백그라운드 진행, 완료 보고만" 패턴을 선호한다. 메인 스레드 점유 없이 PM이 자율적으로 판단·진행하고, 최종 보고에서 (a) 만들어진 것 (b) 사용자 액션 가이드 (c) 한계/후속 작업 (d) 결정 사항을 정리해 한 번에 받는다.

**Why:** 2026-05-17 스플래시 작업 인수인계 시점에 사용자가 명시적으로 "메인 스레드는 점유하지 말아 달라" "완료 보고만 돌려주면 됨" "사용자 추가 결정이 꼭 필요한 새 이슈가 생기면 그것만 보고서에 적시" 라고 운영 지침을 줌. 또한 "기획자한테 검토하라고 해"처럼 PM이 specialist 역할을 직접 흡수하거나 디스패치하는 것은 PM 재량에 맡김.

**How to apply:**
- 작업 진행 중 추가 결정이 꼭 필요한 새 이슈(예: 정책 거부, 사용자 권한 필요한 자산 덮어쓰기)만 보고서에 명시. 그 외는 PM 판단으로 진행.
- 카피라이팅·기획 검토 등 사용자가 "기획자한테 의뢰" 요청하면 specialist 디스패치 도구가 없을 경우 PM이 game-design-planner 관점에서 직접 검토 결과를 제시해도 됨.
- 완료 보고 구조: 만들어진 것 / Unity Editor 액션 / 한계 / 결정한 카피 — 이 네 가지 섹션을 항상 포함.
