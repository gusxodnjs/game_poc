## 0. 목표 (사용자 3지시)
1. **근접 자동 잡기**(포켓몬고식) — 탭이 아니라 내 위치가 생물에 닿으면 → **근접 시 "잡기" 버튼**.
2. **도로↔나무 경계 미화** — 나무가 도로 침범 + 숲→도로 하드 경계 **둘 다** 개선.
3. **피크민 한손 카메라** — **회전(방향)+줌 먼저**, 고도(틸트)는 후속. 한손 드래그=회전으로 **자유 팬 교체**.

3 PR 분해. 신규 에셋 불필요(전부 로직).

## 1. PR-1 · 한손 회전 + 줌 (피크민, 틸트 제외)

### 모델
평면 탑다운 유지 + **맵 루트 회전 + 오브젝트 빌보드 역회전**:
- `TilemapRenderer.headingDeg` 상태(0~360). **`_root.localRotation = Euler(0,0,headingDeg)`** → 지면 타일/오버레이가 통째 회전(도로 방향 바뀜).
- 오브젝트(나무/집/풀/꽃/그루터기/생물)는 자식 스프라이트 `localRotation = -headingDeg` → 위치는 맵과 함께 돌되 **항상 화면 수직**(빌보드).
- 카메라 transform 은 고정(탑다운). 회전은 콘텐츠(_root)에만.

### 제스처 (교체)
- **한 손가락 드래그 = 회전**(가로 이동량 → Δheading). 세로 이동 무시(틸트 후속).
- **핀치(두 손가락) = 줌**(기존 `SetZoom` 유지, clamp 2.5~9).
- **자유 팬 / "내 위치" 버튼 제거.** 카메라는 항상 내 위치(GPS) 중심 → `_centerLat/Lon`은 항상 `_gpsLat/Lon`. `_followGps` 자유둘러보기 분기 제거.

### 커버리지 & 오프셋
- 회전 시 빈 모서리 방지: 타일 산출 범위를 **대각선 반경**(`ceil(sqrt(halfW²+halfH²))+padding`)으로 확장(rangeX=rangeY).
- `PlayerGuiOffset`(GPS-중심 px 오프셋)도 heading 만큼 회전(맵이 돌면 플레이어 표시도 일관).

### 협업 인터페이스
- `TilemapRenderer.HeadingDeg`(public, CreatureField 가 생물 역회전에 사용).
- `CreatureField`: 각 생물 `sr.localRotation = Euler(0,0,-HeadingDeg)`(빌보드).

### 트레이드오프(명시)
- 픽셀 타일을 임의 각도 회전 → 가장자리 계단현상(Point 필터). 부드러운 회전은 2.5D 틸트 라운드에서. PoC 수용.

## 2. PR-2 · 도로↔나무 경계 미화 (`PickObject` 튜닝, 에셋 0)

현재 문제: (a) 가로수가 도로에 바로 붙어 캐노피가 도로 위로 덮임, (b) forest 90% 나무가 도로에 벽처럼 하드하게 끊김.

- **(a) 나무 도로 침범 방지:** 가로수(길/도로 인접 grass 나무)는 **도로가 북쪽(위)에 바로 접한 타일엔 금지**(캐노피가 북으로 뻗어 도로를 덮으므로). 대신 그 가장자리엔 덤불/풀로 완충. 그 외 인접엔 한 칸 물려 배치.
- **(b) 숲→도로 페더링:** 도로/길에 인접한 forest 타일은 **나무 확률 하향**(90%→~45%) + 나머지를 덤불/작은 나무(스케일↓)로 → 가장자리가 점진적으로 옅어짐.
- 결정론(TileHash)·기존 falloff 유지. NearType 로 인접 판정.

## 3. PR-3 · 근접 자동 잡기 버튼 (포켓몬고식)

- **탭 잡기 제거**(`HandleTap`/`TryCatchAt`). 입력은 PR-1 회전에 양보.
- `TilemapRenderer`에 내 GPS 타일좌표 접근자 `GpsTileFrac(out txf,out tyf)`(`_gpsLat/Lon` 기반).
- `CreatureField`:
  - 매 프레임 내 위치 반경 **`catchRangeTiles`(~1.2타일 ≈ 4m)** 안의 가장 가까운 비-flying 생물을 "잡기 대상"으로 선정.
  - 대상 **하이라이트**(부드러운 스케일 펄스 또는 약한 외곽 강조).
  - 대상이 있으면 **하단 중앙 "잡기" 버튼**(IMGUI, safe-area, 엄지 도달) 표시 → 누르면 기존 `flying`(→ 내 행성으로) 연출 + `GameSession.AddCaught` + 카운터/토스트.
  - 대상 없으면 버튼 숨김.

## 4. 에러/폴백
- heading/회전: `_root` null 등은 기존 EnsureInitialized 자가복구로 커버. 회전 적용 전 `_root != null` 가드.
- 근접 대상 없음 → 버튼 미표시(정상).
- 생물 스프라이트/타일 null → 기존 폴백(검은화면 금지).
