# 생동감 라운드 — 맵 자연스러움 + 캐릭터 줌/커스터마이징 + 움직이는 생물

- **이슈:** #52 후속(타일맵 v1 머지 PR #53 = main `f235385` 이후)
- **작성일:** 2026-05-31
- **방향:** [[map-direction-hybrid-rpg]] 하이브리드(실지형 골격 + RPG 미화) 기준 유지. 이번 라운드는 **생동감**(움직임·다양성) 보강.

---

## 0. 목표

배포 피드백 "아직 부자연스럽다 / 좀더 자연스럽고 생동감 있게". 사용자 5개 지시:
1. 맵 다듬기 계속(더 자연스럽고 생동감 있게)
2. 핀치줌 시 캐릭터도 같이 줌인/아웃
3. 나중에 캐릭터 커스터마이징 가능하도록 고려(구조만)
4. 캐릭터 모자 벗기기
5. 샘플 생물 몇 개 배치(움직이는 생물) — **배회 + 탭 잡기 + 행성 전송(기록+카운터)**

독립 배포 가능하므로 **이슈-퍼스트 / PR-당-작업**으로 4개 PR로 분해. 에셋 신규 필요는 모자 제거뿐(디자이너 파이프라인). 나머지는 기존 에셋 활용 dev.

## 1. PR-A · 맵 자연스러움 R2 (`TilemapRenderer` 단독, 신규 에셋 0)

### 진단(코드에서 자명한 어색함 원인)
- 빈 잔디 deco 확률 ~55%가 위치 무관하게 균일 → "노이즈 카펫", 공터/군집 대비 없음.
- 오브젝트 스케일 변형 0(flipX만) → 동일 크기 반복 티남.
- jitter ±0.25타일 → 타일 격자에 정렬되어 보임.
- 꽃이 낱개 산포 → 군락감 없음.

### 개선(전부 `PickObject` / 오브젝트 패스 내 결정론적 해시 튜닝)
- **(A) 인스턴스 스케일 변형:** 오브젝트 SpriteRenderer `localScale`을 해시로 변형. 나무 0.82~1.22, 풀/꽃/덤불 0.85~1.15. (Y-sort·하단앵커 유지되도록 스케일만, 위치 보정 불필요)
- **(B) 클러스터 밀도 변조:** coarse 셀(`tx>>2, ty>>2`) 해시로 0~1 클러스터값 산출(이웃 셀 보간으로 저주파화). 빈-잔디 deco 확률을 이 값으로 변조 → 공터(낮음) ↔ 군집(높음). 평균 밀도는 현행보다 낮춰 숨통.
- **(C) 꽃 클럼핑:** 꽃은 별도 "꽃밭 패치" coarse 셀(다른 해시 시드)이 활성일 때만, 그 안에서 군락 생성. 패치 밖 잔디엔 꽃 거의 없음.
- **(D) jitter 확대:** ±0.25 → ±0.4(범위 가변)로 넓혀 격자 정렬 해소. 단 캐노피가 인접 타일 침범해도 Y-sort로 자연스럽게.

### 불변 계약
- 결정론 유지: 같은 타일좌표 → 항상 같은 결과(`TileHash` 기반). 팬/리프레시해도 깜빡임 없음.
- 숲/길옆 가로수 falloff 로직(`NearType`)은 유지하되 스케일 변형은 함께 적용.

### 테스트
- 순수 로직 헬퍼(클러스터값, 스케일 산출)는 Edit 모드 단위테스트 가능하면 추가. 렌더 결과는 수동(Editor) 검증.

## 2. PR-B · 핀치줌 캐릭터 스케일 연동

현재 `PlayerAvatar`는 고정 스케일(characterScale 3.2 등)로 GUI에 그림 → 맵을 핀치줌(orthographicSize 2.5~9, 기본 4.5)해도 캐릭터 크기 불변 → 월드와 따로 놂.

- `TilemapRenderer`에 `public float ZoomFactor => baseOrtho / mapCamera.orthographicSize;` 노출(baseOrtho=4.5, 즉 기본 줌에서 1.0, 줌인 시 >1).
- `PlayerAvatar`가 캐릭터/정확도 링/그림자 스케일에 `tilemap.ZoomFactor`를 곱함. 줌인하면 캐릭터·링·그림자가 함께 커지고 줌아웃하면 작아짐.
- 클램프: ZoomFactor 범위는 ortho 클램프(2.5~9)에서 0.5~1.8로 자연 한정 — 별도 추가 클램프 불필요(필요시 캐릭터 가독성 위해 하한만).

  - `hat` 필드는 미래 토글 자리(현재 항상 false, 모자 프레임 에셋 없음 — 추가는 후속).
- config 소스: 우선 `PlayerAvatar` 인스펙터 기본값. (저장/로드·선택 UI는 범위 밖.)

## 4. PR-D · 움직이는 샘플 생물 + 잡기 + 행성 전송 (#5, 신규 에셋 0)

### 에셋(기존)
- `assets/sprites/ladybug_anim_frame0~3_64x64.png`, `honeybee_anim_frame0~3_64x64.png` (각 4프레임). `data/species.json`에 movement(`slow_linear`)/speed(ladybug 50, honeybee 더 빠름) 정의.

### 배회(wander)
- 새 컴포넌트 `CreatureField`(HelloScene, 맵 위 레이어). 화면 내 **잔디 타일 위**에 N마리(기본 4~6) 스폰.
- 각 생물: 4프레임 애니(약 8fps), `species.json` 속도로 wander(랜덤 방향 + 가끔 방향 전환; 무당벌레 느린 직선, 꿀벌 더 빠르고 자주 전환). 물/건물 타일은 회피(간단히 잔디/숲/길만 허용).
- 좌표계: 맵 월드 좌표(타일 단위)에 배치 → 팬/줌/GPS 추적에 맵과 함께 움직임(타일 렌더러와 동일 변환 사용). 화면 밖으로 나가면 despawn + 가시영역 가장자리에 리스폰해 항상 ~N마리 유지.
- 줌 연동: PR-B의 ZoomFactor와 동일하게 생물도 줌에 따라 크기 일치(맵 단위 배치라 자동, GUI 아니면 자동).

### 잡기(tap)
- 생물 탭(터치/클릭 → 스크린→월드 히트) 시: "잡힘" 비트(짧은 스케일 팝/반짝) → 화면 코너의 "내 행성" 타겟으로 날아가는 연출(0.5초 lerp) → despawn.
- 입력은 맵 드래그/핀치와 충돌하지 않게: 짧은 탭(이동<임계, 멀티터치 아님)만 잡기로 판정.

### 행성 전송(기록 + 카운터)
- 잡은 생물 저장소 신설(현재 없음). `GameSession`에 행성별 컬렉션:
  - 저장: `PlayerPrefs` 키 `collection_<cellId>` → `species_id:count` 직렬화(예: `ladybug:3,honeybee:1`).
  - API: `GameSession.AddCaught(string speciesId)`, `IReadOnlyDictionary<string,int> GetCollection(cellId)`, 총합 `CaughtTotal`.
- 잡으면 +1 저장 + 토스트("무당벌레 → 내 행성으로!") + 화면 코너에 누적 카운터(예: 🐞×3) 표시(IMGUI).

### 범위 밖
- 행성 위 실제 생물 스프라이트 배치/행성 발전 시각화(다음 라운드).
- 식물 생물(정적)의 잡기, 미니게임, 서버 동기화.
- 기존 `DiscoveryDetection`(셀 교차 알림)과의 통합 — 별개 유지(중복 알림 허용, 후속 정리).

## 5. 에러/폴백
- 생물 스프라이트/타일셋 null → 해당 요소 스킵, 검은화면 금지(기존 정책 계승).
- `PlayerPrefs` 손상/빈 값 → 빈 컬렉션으로 degrade.
- ZoomFactor 분모 0 방지(ortho>0 보장, Awake에서 설정).
