# 작은정복자들 — 시나리오 v2

> **범위**: PoC v7.2 (Unity 6 LTS / iOS) — GPS 시드 시스템 도입 (옵션 C 채택)
> **변경 대상 화면**: 스플래시 → **PlanetIntroScene (IMGUI)** → PlayScene
>   *(v1의 PlanetSelect → PlanetIntroScene 으로 대체. 사용자 선택 행위 없음.)*
> **톤 레퍼런스**: 피크민 블룸 + 스타듀밸리. 따뜻하고 작은 단어. SF 톤 금지. (v1 §0 승계)
> **상위 문맥**: 스플래시 카피 *"걸음마다, 자라나는 세계"* / 시작 버튼 *"첫 걸음 떼기"* 와 운율 통일. (v1 승계)
> **핵심 메타포 (v2 확장)**: *내가 걷는 곳이 내 행성*. 동네마다 다른 원시 행성을 산책으로 회복시킨다.

## 1. GPS 시드 컨셉 도입 *(신규 — PlanetIntroScene 상단 표시)*

> IMGUI Label 한 블록. 2~3문장. 자동 결정 사실을 직접 설명하지 않고 **"여기"** 라는 장소 어휘로 자연스럽게 전달.

```
지금 당신이 서 있는 자리에서,
하나의 작은 세계가 깨어납니다.

같은 자리로 다시 오면, 같은 세계가 당신을 기다립니다.
```

**설계 의도**:
- "GPS로 시드를 만든다"는 시스템 어휘를 텍스트로 노출하지 않음. 톤 가드 ([[feedback-tone-brand]] — SF 어휘 금지).
- 마지막 줄은 영구성(§7) 보장의 시적 표현. 유저가 "내 동네 = 내 행성" 메타포를 한 줄로 습득.
- "여기" / "자리" / "걸음" 등 신체·공간 어휘로 일관.

## 2. 행성 3종 *(v1 §1 100% 승계 — base 정의)*

세 행성은 **모두 원시 단계의 서로 다른 형태**다. v1 §1을 단일 출처(SSOT)로 그대로 인용. **단, 표시 이름은 시드 변주 형식(자동 이름 풀, §5)으로 런타임에 결정**되며, 아래 "base 이름"은 lore·색조·종 풀의 키로만 사용된다.

> **메타포 분리 (v1 승계)**: 시작 종 풀은 "산책지에서 발견할 종"의 슬롯이지, "행성에 이미 사는 생물 목록"이 아니다. 본문 lore에는 종 이름을 직접 등장시키지 않는다.

### 2.1 `volcano` — base: **잠든 화산**
- **base 이름**: 잠든 화산 *(런타임 표시 이름은 §5 자동 이름 풀로 결정. base 이름은 폴백 + 디버그용)*
- **Lore (1~2문장)**:
  검은 용암이 식어 굳고, 갈라진 틈 사이로 흰 김이 가늘게 오른다.
  바위는 아직 따뜻하고, 무언가가 자랄 자리는 비어 있다.
- **시작 종 풀 (4)**: `dandelion` / `foxtail_grass` / `white_clover` / `ladybug`
- **보너스 종 (1)**: `white_clover`
- **색조 톤 권장**: 식은 용암 + 잿빛 적갈 `#5C3B33` (Shader Hue/Tint, 채도 -15, 명도 -5)
  - **시드 변주 (v2 신규)**: hue ±15° 회전 허용 (시드 hash bits 사용). 채도/명도는 base 고정.
- **분위기 키워드**: 검은 바위, 흰 김, 잔열, 정적

### 2.2 `ice` — base: **얼어붙은 평원**
- **base 이름**: 얼어붙은 평원
- **Lore (1~2문장)**:
  회청색 얼음이 지평선까지 깔리고, 바람이 한 번 길게 분다.
  깊은 정적 아래, 무언가가 풀리기를 기다리고 있다.
- **시작 종 풀 (4)**: `dandelion` / `foxtail_grass` / `honeybee` / `ladybug`
- **보너스 종 (1)**: `honeybee`
- **색조 톤 권장**: 새벽 회청 `#6E8AA0` (Shader Hue/Tint, 채도 -20, 명도 +3)
  - **시드 변주**: hue ±15° 회전.
- **분위기 키워드**: 얼음, 정적, 회청, 긴 바람

### 2.3 `desert` — base: **메마른 들녘**
- **base 이름**: 메마른 들녘
- **Lore (1~2문장)**:
  황토색 모래가 끝없이 펼쳐지고, 갈라진 땅 위로 햇빛만 내려앉는다.
  바람 한 결이 어디선가, 작은 무언가를 실어 온다.
- **시작 종 풀 (4)**: `dandelion` / `foxtail_grass` / `cherry_blossom` / `ladybug`
- **보너스 종 (1)**: `cherry_blossom`
- **색조 톤 권장**: 황토 + 메마른 갈색 `#B89968` (Shader Hue/Tint, 채도 -10, 명도 +5)
  - **시드 변주**: hue ±15° 회전.
- **분위기 키워드**: 황토, 갈라진 땅, 햇빛, 마른 바람

## 4. PlanetIntroScene 인트로 카피 *(v1 §2 변형)*

v1은 "세 개의 작은 세계가 ... 천천히 골라 주세요" 형식으로 **선택 의례**를 카피로 담았다. v2는 GPS 자동 결정이므로 **"선택"** 어휘를 제거하고, "여기" "지금" 의 장소 어휘로 톤을 잡는다.

> 위 §1 (GPS 시드 컨셉 도입) 카피가 인트로 카피의 역할을 흡수한다. PlanetIntroScene 상단 인트로는 §1 단락 하나로 충분.

**대체 카피 후보 (PM/QA 선택지 — 1종 채택)**:

| 안 | 카피 | 운율 / 톤 노트 |
|----|------|--------------|
| **A (권장)** | 지금 당신이 서 있는 자리에서, / 하나의 작은 세계가 깨어납니다. / 같은 자리로 다시 오면, 같은 세계가 당신을 기다립니다. | 영구성 보장 한 줄 포함. §1과 동일. **채택**. |
| B | 발 아래, 하나의 작은 세계가 / 당신의 첫 걸음을 기다립니다. | 단순. "걸음" 앵커 강함. 영구성 단서 없음. |
| C | 여기, 한 세계의 첫 새벽이 / 당신을 만납니다. | 시적이지만 "만난다"의 능동성이 게임 액션과 약하게 충돌. |

→ **확정**: **안 A**. §1 본문과 동일하게 사용.

## 5. 자동 이름 풀 *(신규 — v2 핵심 산출물)*

### 5.0 규칙

표시 이름 = **"{형용사} {행성타입별 명사}"**.
deterministic — 시드 hash의 비트로 (형용사 index, 명사 index) 결정. 같은 GPS 셀 → 항상 같은 이름.

**알고리즘 (PM/개발 참고)**:
```
seed_hash = SHA1(lat_floor_001 + "," + lon_floor_001)   # 1km 셀 키
type_idx  = seed_hash[0:1]   % 3         → volcano/ice/desert
adj_idx   = seed_hash[1:3]   % len(ADJ)
noun_idx  = seed_hash[3:5]   % len(NOUN[type])
hue_off   = (seed_hash[5:6] % 31) - 15   → -15° ~ +15°

if (adj_idx, noun_idx) ∈ BLACKLIST[type]:
    noun_idx = (noun_idx + 1) % len(NOUN[type])   # 한 칸 회전. 재충돌은 사실상 0.
```

### 5.1 형용사 풀 (24개)

원시·황량·정적·느린 시간 모티프. 모든 행성타입에 자연스럽게 결합 가능한 어휘로 선별.
한국어 1~3음절 + 받침 어울림 검수.

```

2.  고요한
3.  메마른
4.  얼어붙은
5.  식어버린
6.  잔잔한
7.  흩어진
8.  텅 빈
9.  잠잠한
10. 굳어진
11. 가라앉은
12. 멈춰선
13. 낡은
14. 바랜
15. 희미한
16. 가물거리는
17. 옅은
18. 무거운
19. 느린
20. 깊은
21. 외딴
22. 한적한
23. 비어버린
24. 오래된
```

**검수 노트**:
- "잔잔한"·"고요한"·"잠잠한" 은 의미 유사. 셋 다 유지 — 명사와의 결합 음운/리듬이 미세하게 다름.
- "메마른" / "얼어붙은" / "식어버린" 등 특정 type에 강하게 기우는 형용사 포함 — 의도된 변주 폭.

## 6. 시작 버튼 카피 확정

v1 §4의 "여기로 첫 걸음" 과 PM 1차 안 "여기서 첫 걸음" 비교.

| 안 | 카피 | 의미 | 운율 | 평가 |
|----|------|------|------|------|
| A | 여기**로** 첫 걸음 | 방향: 다른 곳에서 → 여기로 | "걸음" 앵커 + 방향성 강함. v1 자산. | "행성 선택" 의례에 적합. |
| **B (채택)** | 여기**서** 첫 걸음 | 위치: 이미 여기서 시작 | "걸음" 앵커 유지. 위치 고정성 강함. | **GPS 자동 결정 = 이미 여기 있음** 의미와 정확히 정합. |

→ **확정: "여기서 첫 걸음"**.

**근거**:
1. v2 핵심 컨셉이 *내가 서 있는 자리 = 내 행성* 이므로 **이동(로)** 보다 **현재 위치(서)** 의 조사가 의미 정합.
2. §1 본문 "지금 당신이 서 있는 자리에서" 와 "서" 조사가 운율적으로 호응.
3. "걸음" 앵커 단어 유지 → [[feedback-tone-brand]] 규칙 충족.

**Splash 진입 버튼 "첫 걸음 떼기"** 는 v1 그대로 유지 (변경 없음).

## 7. 영구성 정책 *(v1 §5 단순화)*

**결정: deterministic 시드 + `last_seed` 기억**. 선택 행위 자체가 없으므로 v1의 옵션 비교표 불요.

| 동작 | 처리 |
|------|------|
| PlanetIntroScene 최초 진입 | GPS fix 1회 → 1km 셀 key 생성 → 시드 hash → type + 자동 이름 + hue 결정 → 세션 동안 고정 |
| PlanetIntroScene 재진입 (같은 자리) | 같은 셀 key → 같은 hash → 같은 행성 자동 재현 (deterministic이라 `last_seed` 없어도 동일) |
| PlanetIntroScene 재진입 (다른 자리) | 다른 셀 key → 다른 행성. 이전 행성은 폐기되지 않음 (다음에 그 자리 가면 다시 만남) |
| PlayerPrefs `last_seed` 저장 | 부팅 시 GPS fix 실패/지연 시 폴백 표시용. 정상 fix 시엔 GPS 우선. |

**PoC 범위 가드 ([[project-poc-scope]])**: 다중 행성 컬렉션·"내가 다녀온 행성" 도감은 PoC 외. 본 정책은 단일 행성 표시만 보장.

**구현 메모 (PM 참고)**:
- `PlayerPrefs("last_seed", "<셀key>")` 저장. 셀key 포맷: `"{lat_floor_001:.2f},{lon_floor_001:.2f}"` (예: `"37.50,127.03"`).
- GPS 권한 거부/실패 시: `last_seed` 사용 → 그것도 없으면 fallback seed (예: `"0.00,0.00"`)로 base `volcano` 표시. 사용자에게 경고 띄우지 않음 (PoC).

## 9. 자기 검증 체크리스트

- [x] **모든 행성이 원시 상태인가?** — v1 §1 100% 승계. base 3종 모두 빈 자리 / 정적 / 마름 명시.
- [x] **종 풀이 "행성에 사는 생물"로 오독되지 않는가?** — v1 메타포 분리 단서 유지, lore에 종 이름 미등장.
- [x] **자동 이름이 모든 (형용사, 명사) 조합에서 톤 정합인가?** — 블랙리스트 19개 제거. 화이트리스트 15개 시연용으로 우선 노출. 잔여 ~95% 조합도 자연·시간 어휘로 통제.
- [x] **5분 내 읽을 분량인가?** — 본문 약 1.8p, 자동 이름 풀 표 1p (참조용). 카피 단락은 v1과 동일 7개.
- [x] **톤이 SF로 새지 않는가?** — "GPS", "시드", "hash" 등 시스템 어휘는 §1 본문에 노출 0회. §5 알고리즘은 개발자 참고용 박스로만 표시. "행성"·"세계"·"별" 어휘 명사 풀 미포함.
- [x] **"걸음" 앵커 유지?** — §1, §3 (3종 전부), §6 모두 "걸음" 등장.
- [x] **개발자가 추가 질문 없이 구현 가능한가?** — §5 알고리즘 의사코드 + §7 영구성 처리표 + §8 화면 구조표 = 충분.
- [x] **아티스트가 카드 시안 작업 가능한가?** — v1 §1 색조 + §2 시드 변주 (hue ±15°) 명시. base 마스터 1종에 hue 셰이더 오프셋만 적용하면 됨.
